# Changelog

## 0.5.2 - 2026-07-23

- 在根目录与兼容运行时 `SKILL.md` 增加快速开始：提供只要成稿、先评审再改写、减少模板化表达和只审计 Office 文件四类可复制请求；
- 新增打包内的 `references/quickstart.md`，说明最小输入、信息不足时的保守处理、无法核验或读取时的降级边界，以及常见问题；
- 在 README 与快速开始中加入国内 SkillHub 入口；安装提示保留 Gitee 同步镜像作为 GitHub 无法访问时的备选，但不把平台路径、版本或 SHA 细节写死给智能体；
- 为快速开始文件增加结构验证与打包回归，确保发布包中可见、可用。

## 0.5.1 - 2026-07-23

- 新增公开能力清单：在中英文 README 中按规则编号、识别范围、典型信号、处置方式和不误伤边界列出提示词、推理、模型自述、工具残留、公式化转折、技术叠加信号等审查类型；
- 新增 `RW-W-213` 机械枚举信号：在 `public-article` 和 `technical-commentary` 语境中识别三个以上句首排序标记，提示检查“第一、第二、第三”是否掩盖了段落功能单一；
- 对政策条文、研究方法、操作步骤和正式清单保留编号结构，不把排序重复解释为作者身份或 AI 生成结论；
- 更新中文自然表达规则、公众号和技术评论体裁包、根目录与运行时 SKILL，并增加正反例回归测试。

## 0.5.0 - 2026-07-23

- 新增 `technical-commentary` 技术解读/产业评论体裁包，要求区分发布方描述、作者推断和行业预测，并核对技术数字的口径、来源、对象和适用条件；
- 新增组合型模板信号预检：识别短距离内叠加的二元反转、重要性宣告、用户泛化和无条件预测，输出 `RW-W-209` 至 `RW-W-212` 警告；不判断作者身份，不把单个正常对比句列为错误；
- 新增中文技术评论 few-shot 与回归测试，覆盖模型参数、token 激活量、端侧推理和设备条件的事实与限定边界；
- 更新根目录与运行时 SKILL、中文自然表达规则、README 能力说明；每个版本必须先写入本变更日志，再允许版本同步器应用版本号。

## 0.4.0 - 2026-07-22

- 将唯一技术 Skill ID 从 `review-write` 迁移为 `reviewwrite`，减少调用时输入连字符的摩擦；slash 调用统一为 `/reviewwrite`；
- 产品显示名统一为“审写 · ReviewWrite”，将“审写”和“ReviewWrite”定义为自然语言简称，而非多份可安装 Skill；
- 安装器现在检测旧 `review-write` 目录并拒绝并行安装，要求先报告路径、由用户决定迁移或删除，避免重复规则加载；
- 同步 README、安装提示、平台兼容说明、商场元数据、打包资产、Release 工作流、测试与验证规则。

## 0.3.0 - 2026-07-22

- 新增 `Office QA` 按需交付质检路由：DOCX/PPTX 默认只读审计，不静默统一字体、不覆盖原件，也不把工具过程写入正文；
- 新增 `scripts/office_qa.py`：检查直接字体、中文/英文脚本字体、主题引用、确认 profile 不匹配、可选目标字体库存和可选渲染门禁；无模板、字体清单或视觉复核时不伪造“字体合规”结论；
- 新增字体 profile 规范、Office 集成边界、配置样例与最小 OOXML 回归样本，覆盖 DOCX/PPTX 中英文字体不匹配及渲染不可用的阻断边界；
- Release runtime 包现在包含只读 Office QA 脚本；安全说明明确其不联网、不改输入文件，且仅在显式渲染时调用本机渲染工具并写入预览目录；
- README 和运行时文案增加 Office QA 的用户入口、边界、命令与“渲染后逐页人工检查”要求。

## 0.2.1 - 2026-07-22

## 0.2.0 - 2026-07-22

- 为泄漏预检增加体裁与语境感知的豁免注册表（`EXEMPTIONS`），落地能力接口的 `exceptions` 字段：
  - 新增 `--context` 声明被授权语境（`ai-safety`、`prompt-engineering`、`software-docs`、`ai-disclosure`、`dialogue-transcript`、`editorial-audit`），把该语境下合理的硬失败降级为警告，保留人工确认信号而非无差别拦截；
  - 新增 `--genre` 声明体裁（公文、政策、公众号、研究报告、营销），忽略该体裁中功能性的格式警告（结构化加粗条目）；
  - 新增 `--list-profiles` 列出全部 profile 及其放宽规则；文本与 JSON 输出记录 profile 与降级来源（`applied_profile`/`original_severity`）；
- 豁免只放宽表达层面的信号，绝不放宽事实、数字、引用与限定条件的保护；
- 增加 8 项体裁/语境回归测试，覆盖降级、忽略、无关 profile 不放行、优先级、CLI 与未知 profile 拒绝；
- 在 NOTICE、BRAND、compatibility 与 SKILL 元数据中将维护方标注为中财数碳（北京）科技有限公司与中央财经大学人工智能与数字财经研究中心（CUFE/AIDF）。
