# 中文自然表达信号

本文件用于识别模板化、空泛和翻译腔，不用于判断作者身份，也不用于规避 AI 检测。命中一个词不代表有问题；必须结合体裁、上下文、重复频率和证据判断。

## 判断顺序

看到“真正、全面、深度、显著、有效、重要、关键、值得”等词时，先问：

1. 它是否改变了句子的可验证信息？
2. 后面是否给出主体、动作、对象、范围、时间、标准或证据？
3. 删除或具体化后，是否会损失法律、学术、政策或机构语义？

前两项都是否定且没有正式语义时，优先具体化；只满足第一项或第三项时保留并核对语境。

## 常见信号与修复方向

| 信号 | 常见问题 | 优先改成 |
| --- | --- | --- |
| 真正值得、真正重要、真正关键 | 强调判断但没有标准 | 直接写判断标准、对象和依据 |
| 真正改变、深刻影响、全面提升 | 抽象动词缺少主体、对象或范围 | 谁在什么时间对什么对象做了什么，结果多大 |
| 赋能、助力、推动、促进、引领、打造 | 责任主体和实际动作被隐藏 | 具体行动、产出、受益对象和可观察结果 |
| 具有重要意义、释放巨大潜力、迎来新机遇 | 空泛价值判断 | 说明影响、条件、限制和证据 |
| 值得注意的是、需要指出的是、可以看出 | 连接语代替内容 | 删除，或直接写事实/结论 |
| 换句话说、也就是说、简单来说 | 只重复前句，未增加解释 | 直接写被澄清的判断、因果或条件；确有必要释义时可保留一次 |
| 答案很可能不是、最吸引人的地方在于、这也是最危险的地方 | 悬念或戏剧转折代替论点 | 写清“什么不是答案”、具体风险/吸引力是什么、影响谁及其依据 |
| 不仅……而且……、不只是……更是…… | 机械反转或重复强调 | 只有两项确实形成逻辑关系时保留 |
| 第一、第二、第三；首先、其次、最后 | 段首排序标记反复出现，段落功能和语气没有变化 | 将编号改为问题—证据—判断，或用小标题、转折和具体场景区分功能；正式条文和方法步骤可保留 |
| 总的来说、归根结底、由此可见 | 重复总结或越过证据拔高 | 只保留新增结论，并注明适用范围 |

这些短语不是禁词。学术解释中一次必要的释义、政策文本中明确的风险提示、公众号中有真实叙事功能的转折都可以保留；只有它们没有引入新信息、掩盖证据缺口，或连续堆叠时才应改写。

## 叠加式信号（技术解读）

技术公众号或产业评论中，以下组合比单个词更值得复核：

- 两个以上“不是……而是……/未必……而更可能……”出现在相邻句段；
- “这个细节很关键/这比……更重要/拨正方向”等重要性宣告，与反转或总结判断相邻；
- “大多数用户/很多用户”之后直接推导设备需求或行业趋势；
- 参数、激活量、速度、内存等数字出现，但没有统计口径、测量对象、来源或适用条件。

这类命中只生成 `warn`，不判断作者是否使用了 AI，也不要求删掉所有反转句。应当把判断改回机制、指标、场景和限制条件；单个有明确功能的对比可以保留。

## 机械枚举与排序单一化

“第一、第二、第三”本身不是问题。政策条文、研究方法、操作步骤和正式汇报都可能需要稳定编号。需要复核的是：多个段落都以同一组排序标记开头，段落之间没有新增证据、论证功能或语气变化，读者只能看到编号推进。

处理顺序：

1. 先判断每一项是在列事实、证据、判断还是行动；
2. 同一功能的项目合并或改成小标题，不为了“多样性”强行换同义词；
3. 如果是公众号或技术评论，用具体场景、测量结果、限制条件和结论连接段落；
4. 如果是政策、法规或方法步骤，保留规范编号，并在对应体裁下关闭该警告。

预检规则 `RW-W-213` 只在声明 `public-article` 或 `technical-commentary` 时，对三个以上句首排序标记发出警告；它不把编号重复解释为作者身份，也不改变正式文体的结构义务。

## 保留条件

“真正”可以保留在有明确对比、定义或限定的句子中；“显著”可以保留在有统计或测量依据的学术文本中；“必须、应当、严禁”等规范性词语不能为了自然而删除。不得用错别字、随机口语、虚构经历或刻意不完整替代自然表达。
